What is software testing? 1/2

Software testing is a broad term but in general is a process which validate that a software program, application or product:

  • Meets the business and technical requirements that guided it’s design and development
  • Has no bugs/defects/errors
  • Works as expected

What is software testing? 2/2

From technical point of view software testing evaluate a software item (i.e. class, class method or function) to detect differences between given input and expected output.

Furthermore testing assess the quality of the product.

Testing should be done during the software development process.

Who does software testing? 1/2

In the IT industry, large companies have a team with responsibilities to evaluate the developed software in context of the given requirements. In most cases, the following professionals are involved in testing a system within their respective capacities:

  • Software Tester
  • Software Developer
  • Project Lead/Manager
  • End User

Who does software testing? 2/2

In the incremental software development model (for instance in Scrum), testing is performed at the end of every sprint/iteration and the whole application is tested at the end.

Types of software testing.

  • Manual testing In nutshell manual testing means testing without any automated tools or any scripts. Tester/developer takes over the role of an end-user and tests the software to identify any unexpected behaviour or bug.
  • Automate testing This is a process when a user write scripts or use test framework / library to test the product. This is just an automation of manual testing. This kind of testing is used to re-run the test scenarios/ test suits for example whenever user saves his code or push commits to the repository.

Levels of software testing.

  • Functional testing This is a black-box testing which based on the specifications of the software that is to be tested.
  • Non-functional testing This kind of testing involves testing a software from the requirements which are nonfunctional in nature, but important such as performance (load testing and stress testing), security or user interface.

Functional testing 1/2

We can distinguish several types of functional tests:

  • Unit testing The goal of unit testing is to isolate each part of the program and show that individual parts are correct in terms of requirements and functionality.
  • Integration testing / E2E Integration testing is defined as the testing of combined parts of an application to determine if they function correctly. There are two approaches:
    • Bottom-up integration – unit tests are followed by higher level tests
    • Top-down integration – works other way around
    In this group we can also distinguish End-to-end testing which ensure that software works as expected from user’s perpective.

Functional testing 2/2

  • System testing Once all the components are integrated, the application as a whole is tested rigorously to see that it meets the specified Quality Standards.
  • Regression testing The intent of regression testing is to ensure that a change, such as a bug fix should not result in another fault being uncovered in the application.
  • Acceptance testing This is arguably the most important type of testing, as it is conducted by the Quality Assurance Team who will assess whether the application meets the intended specifications and satisfies the client’s requirements.

TDD in software testing 1/3

Test-driven development is a programming methodology that complement test-first approach of Extreme Programming in which developers write tests before implementing a feature or a unit.

TDD in software testing 2/3

TDD is a process that consists of three phases:

TDD in software testing 3/3

  • Write a failing test Write a test that invokes your logic and assert that the correct behaviour is produced.
    • In a unit test, this would be asserting the return value of a function or verifying that a mocked dependency was called as expected
    • In a functional test, this would be ensuring that a UI or an API behaves predictably across a number of actions
  • Make the test pass Implement the minimum amount of code that results in the test passing, and ensure that all other tests continue to pass
  • Refactor the implementation Update or rewrite the implementation, without breaking any public contracts, to improve its quality without breaking the new and existing tests

Summary

  • An early start to testing reduces the cost and time to rework and produce error-free software that is delivered to the client.
  • Diagnosing and fixing errors identified during proper testing should not be interpreted as a time-consuming process but productive activity.
  • No one can claim that even well tested software is 100% bug-free.
  • Testing software provides a natural way to document our software.
  • TDD is time-saving technique and contribute to the quality of a software.
